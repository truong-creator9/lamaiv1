## Purpose

Build each offer in the Money Model in detail — combining:
- **Godfather Strategy:** Packaging, commitments, bonuses, risk reversal → see `references/godfather-strategy.md`
- **Value Equation:** 4 factors that increase/decrease perceived value, value-based pricing, M-A-G-I-C naming

Each offer type is built differently:
- **Main offer** (Attraction + large Core Upsell): Full Godfather Strategy — needs a landing page/sales page
- **Supplementary scripts** (Rollover, Payment Plan, Continuity Bonus...): Short scripts → see `references/script-templates.md`

> - **hero-mechanism.md** from Hero Mechanism Builder (03)

If Money Model is available → list all offers that need to be built:

```
📋 OFFERS TO BUILD:

Attract new customers: [...]
Upsell existing customers: [...]
Retain those who declined: [...]
Recurring revenue: [...]
```

**Decide which offer to build first:**

```
Logical order:
1. Main attraction offer — no new customers without it
2. Largest upsell — generates the most revenue
3. Supplementary scripts — faster to build, do these later

Exceptions:
→ Have existing customers not yet upsold → build upsell script first (money already there)
→ Main offer is good but many decline due to price → build payment plan first
```

State reasons clearly for the user:
> "I'll start with **[offer name]** because [reason]. If you want to build a different offer first — let me know. Otherwise let's begin."

## Result Presentation Guidelines — Required

When outputting final results → always create an HTML visual following ASSP standards:

- Background: `var(--color-background-primary/secondary)`
- Text: CSS variables — do not hardcode colors
- Have it: border-left green `#1D9E75`, tag `#EAF3DE / #3B6D11`
- Missing: border-left red `#E24B4A`, tag `#FCEBEB / #A32D2D`
- Needs attention: border-left orange `#EF9F27`, tag `#FAEEDA / #854F0B`
- Do not include author name — use "Example:" or "Suggestion:"
- Copy + Download .md buttons
